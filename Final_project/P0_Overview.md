# Environmental Data Science — Final Project Selection Guide

Choose one project from the five options below. All projects use Python in Colab Notebooks.

---

## Final Project Score

Your final project score combines two components:

| Component | Points | Scoring |
|-----------|--------|---------|
| **Project Report** | 100 pts | Group score (same for all members) |
| **Final Presentation** | 25 pts | Individual score |
| **Bonus** | up to +5 pts | Included in report score |
| **Total** | **125 pts + bonus** | |

For **undergraduate groups**, the report score is further adjusted by a difficulty multiplier (see below). The presentation score is not multiplied.

> **Final Project Score = (Report Score × Multiplier) + Presentation Score**
>
> For graduate groups: **Final Project Score = Report Score + Presentation Score** (no multiplier)

Each project report is graded out of 100 points. **Undergraduate** groups receive a difficulty multiplier to reward students who take on more challenging work:

## Group Policy

All projects are completed in groups. The following rules apply:

- **Group size:** 3–5 students. No solo submissions.
- **No mixed groups:** Groups must be homogeneous — all-undergraduate or all-graduate.
- **Graduate groups** must choose Project 3, 4, or 5. The difficulty multiplier **does not apply** to graduate groups; their report raw score (out of 100) is used directly.
- **Undergraduate groups** may choose any project (1–5). The difficulty multiplier applies to the report score as listed above.
- Each group submits **one Colab notebook** (.ipynb) with all members' names listed to Canvas.
- Include a **contribution table** at the end of the notebook documenting each member's role and responsibilities.
- **All team members** must participate in the final presentation.

### Custom Projects

Groups may propose a custom project of their own design, subject to instructor approval:

- Submit a **half-page proposal** covering: dataset, analysis methods, expected figures, and why the scope is comparable to one of the existing projects.
- Include a **draft rubric** (modeled on the existing rubrics, totaling 100 points).
- The proposal deadline is **at least 2 weeks before the regular project start** so there is time for review and revision.
- If a custom proposal is not approved, the group must join an existing project in time.
- The instructor will assign a difficulty rating (and corresponding multiplier for undergraduate groups) to approved custom projects.

## General Requirements — All Projects

- Submit a Colab Notebook (.ipynb) with all cells executed and outputs visible.
- Use Markdown cells to label each section clearly, matching the steps in the project guide.
- Every figure must have a descriptive title, axis labels with units, and a legend where needed.
- Cite all data sources (provider, product name, version, access date).
- Code must run top-to-bottom without errors when cells are re-executed in order.
- Written analysis must be in your own words. AI coding assistants are permitted, but you must be able to explain all code you submit — instructors may ask during grading.
- **Late policy:** No late submission will be accepted.
